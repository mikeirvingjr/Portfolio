Game Development

Here, we will take a look at the Game Engines I have used or attempted to use

MonoGame

MonoGame is a managed C# framework for developing cross platform games. It works for a multitude of platforms such as Windows,  MacOS,  Linux,  Android,   and iOS to name a few. If licensed, it will work for Xbox, PS4 | PS5, and Nintendo Switch consoles.

I first used MonoGame somewhere around 2010-2012 where I started porting my old Monopoly C++ code I had written for a Software Engineering class for my Bachelor's degree in Computer Science. Back then, the game was written in C++ and OpenGL. MonoGame proved that it was easier to write in C# and let the framework worry about the graphics layer. I was also using the project to learn C#, but that's is for another story.

Unity

Unity is a 2D and 3D gaming engine designed for professional game development. It supports Windows, MacOS, and WebGL platforms natively, while the game consoles like Nintendo Switch, Xbox, and Playstation need a higher tier and licenses from the respective companies.

I truly did not start my Unity journey until 2023. Did not seem like it was an engine that I would like. Or so I thought. I love coding in C# and little did I know, the scripting language for this engine is solely that.

Unreal Engine

Unreal Engine is a powerful and robust game engine for 2D and 3D games. The primary scripting language is C++. Not for the faint of heart developers.

I have dabbled in Unreal Engine numerous times. I was fascinated by Unreal Tornament and when the first Gears of War game released, I was hooked on what the Unreal Engine could do. Not to mention that I was a C++ developer back then.

Godot

Godot is an open source and game engine that works to create 2D and 3D games. The primary scripting language is GDScript a language similar to python; however, with the MONO framework and later .NET6, C# became supported.

I have only used the C# version of Godot in both v3 and v4. This was partly because of Unity's new price structure, and I was also curious in how much time it would take to port a Unity game to Godot.
